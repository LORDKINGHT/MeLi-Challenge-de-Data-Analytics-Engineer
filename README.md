
# Proyecto: MeLi-Challenge-de-Data-Analytics-Engineer

[![License: MIT](https://img.shields.io/badge/License-MIT-green.svg)](https://opensource.org/licenses/MIT)

## Descripción

Este repositorio contiene la propuesta técnica desarrollada para el MeLi Challenge de Data Analytics Engineer, articulada en un cuaderno Jupyter (`.ipynb`) que incluye análisis, modelos, decisiones de diseño y artefactos de validación. El objetivo es presentar de forma reproducible la solución planteada, su justificación técnica y los resultados obtenidos.

## Objetivo

Dar visibilidad y trazabilidad técnica a la propuesta mediante:
- Documentación clara del problema y solución.  
- Implementaciones ejecutables dentro de Jupyter.  
- Estándares de reproducibilidad.  
- Facilitar revisión, extensión y despliegue por parte de terceros técnicos.

## Alcance

Incluye:
- Desarrollo y explicación de la lógica de la propuesta.  
- Código de análisis y modelado.  
- Supuestos y métricas consideradas.  
- Recomendaciones operativas.  
- Artefactos listos para evaluación técnica (outputs, gráficos, conclusiones).

## Requisitos

- Python 3.10 o superior.  
- Paquetes listados en `requirements.txt` (se recomienda entornos virtuales).  
- Acceso a Jupyter Notebook o JupyterLab.  
- Git para control de versiones (opcional, para quienes clonen el repositorio).

## Clonar el repositorio
git clone https://github.com/LORDKINGHT/MeLi-Challenge-de-Data-Analytics-Engineer.git
cd MeLi-Challenge-de-Data-Analytics-Engineer

Ejemplo de layout esperado:


## Uso

1. Activar el entorno virtual.  
2. Abrir el cuaderno Jupyter:

```bash
jupyter notebook Propuesta_Meli_Julio_Parra.ipynb
```

3. Ejecutar las celdas en orden, revisando secciones como:
   - Introducción y contexto.  
   - Preparación de datos.  
   - Modelado o propuesta técnica.  
   - Resultados y conclusiones.

## Contribuciones

Se aceptan contribuciones bajo los siguientes lineamientos:

1. Hacer fork del repositorio.
